Wallet UI ships two React Native SDK variants. The setup flow is mostly the same, so the docs are shared here and call out the places where the package APIs diverge.

## Choose your package

Choose the package that matches the Solana client stack already used by your app.

<Tabs syncKey="rn-sdk">
	<TabItem label="Kit">
		Use `@wallet-ui/react-native-kit` if your app is built around `@solana/kit` and you want the Kit-native RPC client, `Address` types, and transaction helpers.

		<PackageManagerTabs
			npm="npm install @wallet-ui/react-native-kit"
			pnpm="pnpm add @wallet-ui/react-native-kit"
			bun="bun add @wallet-ui/react-native-kit"
		/>
	</TabItem>
	<TabItem label="Web3.js">
		Use `@wallet-ui/react-native-web3js` if your app already uses `@solana/web3.js`, `Connection`, `PublicKey`, and the web3.js transaction model.

		<PackageManagerTabs
			npm="npm install @wallet-ui/react-native-web3js"
			pnpm="pnpm add @wallet-ui/react-native-web3js"
			bun="bun add @wallet-ui/react-native-web3js"
		/>
	</TabItem>
</Tabs>

The current starter templates are documented for `@wallet-ui/react-native-web3js`.

<Aside type="note" title="Kit projects">
	The Astro docs do not currently advertise parallel starter templates for `@wallet-ui/react-native-kit`. Use the shared Expo guide below to add Kit to a new or existing app.
</Aside>
